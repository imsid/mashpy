# Security Policy

## Supported versions

Mash is pre-1.0 and ships from `main`. Security fixes land in the latest
released version on PyPI. Older `0.x` releases are not patched — upgrade to the
latest version to receive fixes.

| Version | Supported          |
|---------|--------------------|
| Latest `0.x` release | :white_check_mark: |
| Older `0.x` releases | :x:                |

## Reporting a vulnerability

Please do **not** open a public issue for security problems.

Report privately through GitHub's
[Security Advisories](https://github.com/imsid/mashpy/security/advisories/new)
("Report a vulnerability"). This opens a private channel with the maintainers.

When reporting, include as much of the following as you can:

- A description of the vulnerability and its impact
- Steps to reproduce, or a proof of concept
- Affected version(s) and environment
- Any suggested fix or mitigation

## What to expect

- We aim to acknowledge a report within **3 business days**.
- We'll work with you on a fix and coordinate a disclosure timeline.
- We'll credit you in the advisory unless you prefer to stay anonymous.

Thanks for helping keep Mash and its users safe.
